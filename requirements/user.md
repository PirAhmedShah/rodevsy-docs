# User Requirements

## 1. Developer Requirements
* **UR-001 (Reputation Building):** The system must provide a public-facing profile that displays verified skill tiers, project history, and user ratings to establish professional credibility.
* **UR-002 (Payment Security):** The system must provide a "Funds Secured" notification immediately upon contract acceptance, allowing the developer to begin work without fear of non-payment.
* **UR-003 (Dispute Recourse):** The developer must be able to initiate a formal dispute process if a client refuses to approve deliverables despite them meeting the agreed-upon project specifications.
* **UR-004 (Portfolio Verification):** The system must allow developers to link live Roblox Place IDs and upload work samples that are automatically tagged by skill, ensuring their portfolio is verifiable by potential clients.
* **UR-005 (Communication Integrity):** The developer requires a professional, internal communication channel tied to specific projects to prevent scope creep and maintain a clear audit trail of project requirements.



## 2. Project Client Requirements
* **UR-006 (Talent Verification):** The client must be able to filter and search for developers based on verified Skill Tiers, "Champion" status, and historical review scores to minimize hiring risk.
* **UR-007 (Milestone Oversight):** The system must provide a dashboard for the client to view project status, track deliverables, and manage payments across defined milestones.
* **UR-008 (Financial Protection):** The client requires that funds placed into escrow remain protected and are only released upon their explicit approval or automated milestone verification.
* **UR-009 (Project Structuring):** The client must be able to post detailed projects with mandatory fields (budget, deadline, skill tags) to ensure they receive high-quality, relevant proposals.
* **UR-010 (Content Safety):** The client requires assurance that any assets or scripts purchased through the Marketplace have been automatically scanned for backdoors or malicious code before the purchase is finalized.



## 3. General Platform Requirements
* **UR-011 (Identity Trust):** All users must be able to verify the identity of the other party via Roblox OAuth, ensuring that collaboration occurs between legitimate, active community members.
* **UR-012 (Financial Simplicity):** All users require an intuitive wallet interface that displays balances in Credits (USDT parity) and clearly outlines any service fees before transaction execution.
* **UR-013 (Community Engagement):** Verified users must have access to a moderated Forum to seek technical help, discuss development trends, and collaborate on community-wide Roblox initiatives.
